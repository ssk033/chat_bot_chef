1. PROJECT OVERVIEW
--------------------------------------------------------------------------------

  • Product name (UI): Meal-IT!!
  • NPM package name: chbot
  • Purpose: A meal-planning and nutrition web app with an AI “Chef” chat,
    meal-plan flows, a food-photo analyser (Food Tracker), a simple nutrition
    log, and a floating site-guide assistant. Recipe exploration uses a
    PostgreSQL-backed catalogue with optional embeddings for similarity.

  • Credits (as shown in-app): Built by Sanidhya, Rajnish, Sachet, Aayush.

2. TECH STACK
--------------------------------------------------------------------------------

  Framework & language
  • Next.js 16 (App Router), React 19
  • TypeScript 5
  • Node.js runtime for API routes (explicit `nodejs` where required)

  Styling & UI
  • Tailwind CSS v4 (`@import "tailwindcss"` in `app/globals.css`)
  • `@tailwindcss/postcss`
  • Design tokens via CSS variables (`--background`, `--foreground`,
    `--surface`, `--accent`, `--muted-text`, etc.) in `app/globals.css`
    with `.dark` class on `<html>` for theme switching
  • Component helpers: `clsx`, `tailwind-merge` (`cn()` in `lib/utils.ts`)
  • Icons: `@tabler/icons-react`
  • Motion / animation: `motion` (e.g. `motion/react` in file upload UI)

  Data & backend integration
  • PostgreSQL via Prisma ORM (`prisma/schema.prisma`)
  • `DATABASE_URL` — Neon / Postgres connection string
  • Recipe embeddings column uses Postgres `vector` type (`Unsupported` in Prisma)
  • Optional external ML / Food AI:
      - HTTP service default `http://127.0.0.1:8788` — overridden by
        `FOOD_AI_SERVICE_URL`
      - Started locally via `npm run food-ai:dev` (Python uvicorn under
        `ml-models/food-ai-server/` when present)
  • Google Generative AI (`@google/generative-ai`):
      - Chef-adjacent flows, meal planner `/api/query`, site guide
      - Food Tracker refinement via `lib/gemini-food-vision.ts`
      - Env: `GEMINI_API_KEY`, optional `GEMINI_MODEL`

  Client-side persistence (browser)
  • Auth profile: `localStorage` key `meal_it_user` (`lib/client-auth.ts`)
  • Chef chat: anonymous user key + sessions/messages in Postgres;
    header `x-chef-anonymous-key` (`lib/chef-auth.ts`)
  • Theme: `localStorage` key `chef-theme`, boot script in `app/layout.tsx`
  • Meal-plan AI saved drafts: `lib/meal-plan-ai-saves.ts`
      - Legacy key `meal_it_saved_plans`; versioning helpers for planner chat
  • Food Tracker history (thumbnails + last result snapshot):
      - `lib/food-tracker-history.ts` → key `meal-it-food-tracker-history-v1`

  Tooling
  • ESLint 9 + `eslint-config-next`
  • Build: `next build` (Turbopack as per Next 16 defaults)
  • Fonts: `next/font/google` — Geist Sans & Geist Mono (`app/layout.tsx`)

  Optimisations (Next config)
  • `experimental.optimizePackageImports` for `@tabler/icons-react`, `motion`
  • `poweredByHeader: false`
  • Image optimisation: AVIF/WebP, cache TTL, `remotePatterns` for Unsplash

6. API ROUTES (SERVER)
--------------------------------------------------------------------------------

  POST /api/food-ai/predict
  • Multipart image → proxies to Python Food AI `POST {UPSTREAM}/predict`
    (`FOOD_AI_SERVICE_URL` or default localhost:8788).
  • Reads CNN confidence from flexible keys: confidence, softmax_confidence,
    score, cnn_confidence, prob, probability.
  • Rule: if CNN confidence ≥ 0.65 → return CNN JSON (sanitized fields).
    If < 0.65 → call Gemini vision (`lib/gemini-food-vision.ts`) on same image;
    if Gemini succeeds, return dish/macros/confidence from Gemini (CNN demo
    hints preserved where relevant).
  • If CNN unreachable or non-OK HTTP / bad JSON → tries Gemini-only fallback.
  • Requires `GEMINI_API_KEY` in deployment env for cloud without local CNN.

  POST /api/food-tracker/reviews
  • Persists `FoodAiPredictionReview` rows (Prisma).

  POST /api/query
  • Large route: recipe RAG / Gemini-powered replies, embedding generation
    (`lib/embedding-model.ts`), Prisma recipe queries. Used by Chef-related
    flows and meal intelligence.

  GET/POST /api/sessions
  GET      /api/sessions/[sessionId]
  POST     /api/sessions/[sessionId]/messages
  • Anonymous chef sessions backed by Prisma `User` + `ChatSession` +
    `ChatMessage`; keyed via `x-chef-anonymous-key`.

  POST /api/site-guide
  • Site-wide floating assistant (`SiteGuideAssistant`): Gemini + pathname +
    message history (`lib/site-guide-gemini.ts`, `lib/site-guide-prompt.ts`).

  GET/POST /api/user
  POST /api/user/bootstrap
  • User bootstrap / anonymous identity alignment for chef flows.

7. KEY FEATURES — HOW THEY WORK
--------------------------------------------------------------------------------

  7.1 Authentication (lightweight)
  • Not OAuth-heavy in client auth: email stored under `meal_it_user`.
  • Protected pages gate with `getStoredUser()` + redirect to login with `next`.

  7.2 Theme
  • `ThemeToggle` flips `document.documentElement.classList` (`dark`).
  • Initial theme from localStorage + `prefers-color-scheme` (inline script).

  7.3 Chef chat
  • Client loads/creates anonymous key, syncs sessions from API.
  • Messages rendered with user initials avatar vs Chef avatar.
  • Composer: attachments stub; mic; send; TTS play/stop.

  7.4 Meal planner
  • Create form builds URL params → `/meal-plan/chat` seeds first `/api/query`
    turn with structured prompt from intake.
  • Subsequent sends POST JSON `{ message }` to `/api/query`.
  • “Save latest reply” appends to localStorage list for reuse.

  7.5 Food Tracker
  • Client-only preview blob URL; FileUpload component (dropzone + motion).
  • POST FormData `image` to predict API; displays metrics + confidence bands.
  • Review modal POSTs structured feedback (no image stored server-side per copy).
  • History: compressed JPEG thumbnails in localStorage, capped list length.

  7.6 Nutrition tracker
  • Pure React state; totals computed with `useMemo`; no server persistence.

  7.7 Site Guide (global widget)
  • Lazy-loaded client bundle (`site-guide-assistant-root.tsx` dynamic import,
    `ssr: false`) to reduce initial JS.
  • Floating launcher bottom-right; minimise / close; Escape closes.
  • Gemini-backed `/api/site-guide`; welcome message stored client-side with
    stable message IDs.

  7.8 Recipe / embedding pipeline (backend)
  • `lib/embedding-model.ts`: optional local Python script inference OR
    Hugging Face Inference API fallback; caching & timeouts.
  • Turbopack: `path.join(/* turbopackIgnore: true */ process.cwd(), ...)` to
    avoid tracing entire repo during NFT.

10. DEPLOYMENT NOTES
--------------------------------------------------------------------------------

  • Vercel / Node hosts: set `DATABASE_URL`, `GEMINI_API_KEY`.
  • Local CNN (`127.0.0.1:8788`) is NOT reachable from cloud → predict route
    falls back to Gemini vision when key present; host CNN separately and set
    `FOOD_AI_SERVICE_URL` if full CNN + 65% gating is required in production.
  • Ensure Postgres supports `vector` extension if embeddings are used.
  • `npm run build` should pass before ship; `optimizePackageImports` is on.
